# ai-tp-os 项目需求分析 + 优先级

**分析时间**: 2026-06-22 12:00 (UTC+8)  
**分析者**: QClaw Agent

---

## 📝 项目概述

**AI-TP OS** 是一个"AI原生去中心化操作系统"，目标是：
- 以底层系统为本体，整合应用层的 AI 原生去中心化操作系统
- 支持 100+ LLM 应用
- 本地推理优先（Ollama）
- 统一存储层（Agent 记忆共享）
- P2P 网络层（节点自动发现）

**当前状态**: 基础框架已搭建，但功能不完整，依赖问题多。

---

## 🎯 项目最需要什么 (优先级排序)

### 🔴 第一优先级：让核心功能能运行

#### 1. 安装最小依赖
- **需求**: 当前无法 `pip install`（容器环境，无 pip 权限）
- **影响**: 无法运行任何应用
- **复杂度**: 低（但需要 pip 权限）
- **解决方案**: 
  - 创建 `requirements-minimal.txt`
  - 只安装核心依赖（ollama, sqlite3, urllib）
  - 使用 venv 或 `--user` 安装

#### 2. 启动 Ollama 服务
- **需求**: 本地推理核心，所有 Agent 都依赖它
- **影响**: 无法运行任何 AI 功能
- **复杂度**: 中（需要下载 Ollama 二进制文件）
- **解决方案**:
  - 下载 Ollama 二进制文件到 `~/.ollama/`
  - 启动 `ollama serve`
  - 下载模型 `ollama pull qwen2.5:0.5b`

#### 3. 完善 CLI 工具
- **需求**: 让用户可以运行应用
- **影响**: 无法使用系统
- **复杂度**: 低
- **解决方案**:
  - 修复 `scripts/cli.py`
  - 添加 `run` 命令
  - 添加 `status` 命令
  - 添加 `model` 命令

---

### 🟡 第二优先级：完善核心功能

#### 4. 完善 NetworkManager - P2P 功能
- **需求**: 实现节点发现、NAT 穿透、Agent Mesh
- **影响**: 无法去中心化通信
- **复杂度**: 高（需要网络编程）
- **解决方案**:
  - 实现 mDNS 节点发现（使用 `zeroconf` 库）
  - 实现手动节点配置（JSON 文件）
  - 实现 Agent 通信协议（JSON-RPC）

#### 5. 添加更多 Starter Agents
- **需求**: 扩展应用层，从 3 个到 10+ 个
- **影响**: 功能演示不足
- **复杂度**: 中
- **解决方案**:
  - 创建 `ai_chat_agent`（通用聊天）
  - 创建 `ai_code_agent`（代码生成）
  - 创建 `ai_translate_agent`（翻译）
  - 创建 `ai_summarize_agent`（总结）

---

### 🟢 第三优先级：高级功能

#### 6. 实现 RAG 功能
- **需求**: 文档检索增强生成
- **影响**: 无法处理文档
- **复杂度**: 高
- **解决方案**:
  - 集成 `chromadb` 或 `faiss-cpu`
  - 实现文档上传、分块、嵌入、检索

#### 7. 实现 Agent Mesh
- **需求**: 多 Agent 协作
- **影响**: 无法复杂任务分解
- **复杂度**: 很高
- **解决方案**:
  - 使用消息队列（Redis）
  - 实现 Agent 通信协议
  - 实现任务分配和结果汇总
